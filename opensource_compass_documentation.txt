================================================================================
                    OPENSOURCE COMPASS - MASTER DOCUMENTATION
================================================================================

1. EXECUTIVE SUMMARY
--------------------------------------------------------------------------------
OpenSource Compass is an advanced, enterprise-grade AI mentorship application designed to help developers seamlessly explore, understand, and learn from open-source Python codebases. By providing a GitHub URL, the application autonomously clones the repository, parses the entire codebase using Abstract Syntax Trees (AST), generates a dynamic visual dependency flowchart, and stores semantic code embeddings in a vector database. Users then interact with an AI Mentor (powered by Google Gemini 3 Flash Preview) that possesses full contextual knowledge of the ingested codebase using Retrieval-Augmented Generation (RAG).

2. THE PIPELINE: END-TO-END DATA FLOW
--------------------------------------------------------------------------------
The system operates on a highly optimized, dual-phase pipeline: The "Ingestion Phase" and the "Interactive Phase".

A. THE INGESTION PHASE (Triggered by the Landing Page)
1. User Input: The user pastes a GitHub URL into the React frontend and clicks "Analyze". An estimated logarithmic progress bar begins filling the button.
2. Shallow Clone (/ingest): The FastAPI backend receives the URL and uses `GitPython` to perform a "shallow clone" (depth=1). This downloads only the latest snapshot of the code into a hidden local `.temp_repos/` directory, preventing massive history downloads and taking only seconds.
3. AST Parsing & Chunking (/embed): 
   - The backend recursively crawls `.temp_repos/` for all `.py` files.
   - It feeds each file into `tree-sitter` (an Abstract Syntax Tree parser) which intelligently slices the code into logical semantic "chunks" (e.g., keeping entire functions and classes intact rather than cutting blindly by line numbers).
4. Vector Embedding:
   - CPU Vectorized Batching: The backend extracts all chunk strings for a given file and passes them as a batch list to the `SentenceTransformers` AI model (`all-MiniLM-L6-v2`). The model utilizes parallel CPU processing to instantly convert the text strings into 384-dimensional mathematical arrays (vector embeddings).
   - Network Batching: The backend packages all the chunks and their vectors into a single JSON payload and makes exactly one HTTP POST request to the Supabase Postgres Database, inserting the batch into the `code_chunks` table.
5. Transition: The frontend progress bar snaps to 100%, and the user is transitioned to the Dashboard View.

B. THE INTERACTIVE PHASE (The Dashboard)
1. Graph Visualization (/diagram):
   - Upon loading, the Dashboard requests the architecture map.
   - The backend's `graph_builder.py` uses AST to extract all internal `import` statements across the repository.
   - `NetworkX` computes a directed dependency graph.
   - `diagram_generator.py` sanitizes this graph and translates it into a strict `Mermaid.js` syntax string (Graph LR), removing massive file paths to keep node labels clean.
   - The React frontend receives the string and natively renders it as an interactive, draggable SVG flowchart on the right-side canvas.
2. AI Mentor Chat (/chat):
   - The user types a question (e.g., "How does the authentication work?") into the chat panel.
   - The backend converts the question into a vector embedding using the same `SentenceTransformer` model.
   - Dual-Strategy Vector Search: The system queries the Supabase `pgvector` database to find code chunks mathematically similar to the question. It first attempts an RPC similarity search, then falls back to a direct table query filtered by `repo_name` to guarantee the AI only sees code from the exact repository being analyzed.
   - LangChain LCEL RAG: The retrieved code chunks are injected into a LangChain system prompt template as "context", along with the user's question.
   - Generation: The prompt is sent to `Google Gemini 3 Flash Preview` which reads the raw code context and streams back a highly accurate, codebase-specific answer.

3. TECHNOLOGY STACK & CORE PROCESSES
--------------------------------------------------------------------------------
- FRONTEND:
  - React.js & Vite (Lightning-fast HMR and bundling)
  - CSS3 (Vanilla CSS for a bespoke, dynamic dark-mode styling and glowing animations)
  - Mermaid.js (For dynamic, client-side SVG flowchart rendering)
  - Lucide React (For crisp SVG iconography)
  - api/client.js (Decoupled fetch API layer)

- BACKEND CORE:
  - FastAPI (High-performance asynchronous Python web framework)
  - Uvicorn (ASGI web server using `watchfiles` to ignore `.temp_repos/` and prevent crash-loops)
  - Pydantic (For strict data shape validation in `models/schemas.py`)

- PARSING & DATA PROCESSING:
  - GitPython (For autonomous repository cloning)
  - tree-sitter & tree-sitter-python (For Abstract Syntax Tree parsing and chunking)
  - NetworkX (For building and traversing directed dependency graphs)

- AI & MACHINE LEARNING:
  - Embedding Model: `sentence-transformers/all-MiniLM-L6-v2` (Running locally in memory for blazing fast 384-dimensional dense vector generation). The model is "lazy-loaded" to prevent slow Uvicorn startup times.
  - LLM: `Google Gemini 3 Flash Preview` (Google's latest lightweight, multimodal model used for rapid code explanation and RAG generation).
  - Orchestration: `LangChain` Expression Language (LCEL) used to seamlessly chain prompts, retrievers, and LLMs.

- DATABASE:
  - Supabase PostgreSQL
  - pgvector extension (For computing Cosine Similarity between the user's question vector and the code chunk vectors in milliseconds).

5. STABILITY & RESILIENCE MECHANISMS
--------------------------------------------------------------------------------
- Error Sanitization: The `utils/errors.py` middleware wraps FastAPI endpoints. If an external service like Supabase goes down, it catches massive raw HTML traceback errors (like Cloudflare 521s) and converts them into single-sentence, user-friendly UI warnings.
- UI Progress States: The frontend utilizes robust `disabled` states, loading spinners, and an internal logarithmic `setInterval` progress bar. This ensures users cannot spam endpoints and always have visual feedback during heavy background tasks.
- Fallback Nodes: If a user ingests a repository that contains zero python files (or has no inter-file dependencies), the backend `diagram_generator` automatically returns a fallback node `[No Python Architecture Detected]` to prevent the Mermaid.js parser from fatally crashing the React UI.
